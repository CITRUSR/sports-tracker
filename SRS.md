## Система учета спортивных результатов

---

# 1. Введение

## 1.1 Цель

Документ описывает требования к веб-приложению для учета спортивных результатов.

Система предназначена для:
- хранения результатов пользователей;
- просмотра истории тренировок;
- расчета базовой статистики;
- отображения прогресса показателей.

---

# 2. Общее описание

## 2.1 Перспектива продукта

Приложение является веб-системой и работает в современных браузерах.

## 2.2 Функции продукта (кратко)

- Регистрация пользователей  
- Управление профилем  
- Управление и хранение результатов тренировок  
- Просмотр истории тренировок  
- Построение графиков прогресса  
- Вывод статистики  

## 2.3 Ограничения

- Система должна работать в современных браузерах.

## 2.4 Предположения и зависимости

- Пользователи имеют доступ к интернету.

---

# 3. Требования к системе

## 3.1 Функциональные требования

### FR-01 Регистрация пользователя

Пользователь может зарегистрироваться, указав логин и пароль.

---

### FR-02 Управление профилем

Пользователь может заполнить и редактировать профиль.

Профиль включает:
- имя;
- текущий вес;
- дата рождения.

---

### FR-03 Ввод результатов тренировки
Пользователь может добавить тренировку со следующими параметрами:

Упражнения - включают в себя :
  - тип (Пользователь выбирает тип упражнения из заранее предложенного списка.
        При необходимости пользователь может добавить новый тип упражнения, если его нет в списке);
  - количество повторений (если применимо);
  - вес (если применимо);
  - дистанция (если применимо);
- дата тренировки;
- время начала тренировки;
- время окончания тренировки;
- комментарий (опционально).

---

### FR-04 Редактирование тренировки

Пользователь может изменить ранее добавленную тренировку.

---

### FR-05 Удаление тренировки

Пользователь может удалить свою тренировку.

---

### FR-06 Просмотр истории тренировок

- Пользователь может просматривать список своих тренировок.
- Возможна фильтрация по типу упражнения.
- Возможна фильтрация по дате.

---

### FR-07 Построение графиков прогресса по упражнениям

- Пользователь может просматривать график прогресса по выбранному типу упражнения. 
(что считается прогрессом и насколько сильно?)

---

### FR-08 Вывод базовой статистики

Система должна отображать:
- количество тренировок за выбранный период;
- среднюю длительность тренировок за выбранный период.

---

### FR-09 Просмотр прогрессии веса

- Пользователь может просматривать график изменения своего веса за выбранный период.

## 3.2 Нефункциональные требования

### NFR-01 Производительность
Система должна быстро работать

### NFR-02 Надежность
Данные пользователей должны сохраняться в базе данных и не теряться при перезагрузке сервера.

### NFR-03 Безопасность
Доступ к данным пользователя должен быть возможен только после авторизации.

### NFR-04 Совместимость
Система должна корректно работать в современных браузерах:
- Google Chrome
- Mozilla Firefox
- Microsoft Edge

### NFR-05 Удобство использования
Интерфейс должен быть интуитивно понятным и позволять пользователю добавить тренировку не более чем за несколько действий.
